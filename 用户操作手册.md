# 使用手册

## 默认启动方式

进入项目目录后，运行：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run.sh
```

默认模式是：

- `Command + Shift + K` 按一次开始录音
- 再按一次结束录音并执行

## 其他启动方式

- `./run_push_to_talk.sh`
  - 和默认模式一样
- `./run_hotkey.sh`
  - `Command + Shift + K` 触发固定 4 秒录音
- `./run_voice_loop.sh`
  - 按 `Enter` 开始录音

## 常见问题

### 1. 只打开 App，没有继续操作

先看终端里有没有这类报错：

```text
No active window or root filtered out
```

如果有，说明不是命令本身有问题，而是手机上的 Portal 暂时读不到当前界面。

先执行修复：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./fix_portal.sh
```

然后等 10 到 30 秒，再重新运行：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run.sh
```

### 2. 为什么会出现 `FP16 is not supported on CPU; using FP32 instead`

这是本地 Whisper 在 CPU 上转写时的提示，不是报错。

含义是：

- 你的电脑现在用 CPU 跑语音识别
- CPU 不支持 FP16
- Whisper 自动改用 FP32

一般不影响使用，只是会慢一点。

### 3. 手机上的密集辅助框怎么隐藏

可以隐藏。

一键隐藏：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./hide_overlay.sh
```

一键重新显示：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./show_overlay.sh
```

如果你想用键盘热键切换显示/隐藏，先启动：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run_overlay_hotkey.sh
```

然后在 Mac 上按：

- `Command + Shift + O`

就可以切换 Portal overlay 的显示和隐藏。

## 轻量记忆与 heartbeat

项目现在已经有一套轻量版记忆：

- `memory/facts.jsonl`
  - 存稳定事实
- `memory/daily/`
  - 存每天的互动记录
- `memory/heartbeat/latest.md`
  - 存最近一次 heartbeat 摘要

手动生成 heartbeat：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run_heartbeat.sh
```

## 重要教训

- 如果 DroidRun 失败，先不要急着改命令解析逻辑。
- 先确认是不是 Portal 读不到当前 UI。
- 这个项目里的本地 wrapper 只应该做“薄兜底”，不应该替代 DroidRun 自己的任务规划逻辑。
- 最优先的排查顺序是：
  - 先看有没有 `No active window or root filtered out`
  - 有的话先跑 `./fix_portal.sh`
  - Portal 恢复后，再重新执行原始命令
