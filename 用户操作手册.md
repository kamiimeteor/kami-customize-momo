# 使用手册

## 默认启动方式

进入项目目录后，运行：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run.sh
```

默认模式是：

- `Command + Shift + K` 按一次开始录音
- 再按一次结束录音并执行

## 其他启动方式

- `./run_push_to_talk.sh`
  - 和默认模式一样
- `./run_hotkey.sh`
  - `Command + Shift + K` 触发固定 4 秒录音
- `./run_voice_loop.sh`
  - 按 `Enter` 开始录音

- Gemini / OpenAI key 是否真的被当前 shell 看见
- Portal、无障碍、当前前台 App 是否可读

## 常见问题

### 1. 只打开 App，没有继续操作

先看终端里有没有这类报错：

```text
No active window or root filtered out
```

如果有，说明不是命令本身有问题，而是手机上的 Portal 暂时读不到当前界面。

先执行修复：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./fix_portal.sh
```

然后等 10 到 30 秒，再重新运行：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run.sh
```

也可以直接做一次诊断：

```bash
cd "$HOME/Documents/AI_projects/momo-droidrun" && ./run_doctor.sh --app 微信 --open-app
```

如果输出里出现 `likely_unreadable=True`，说明手机上的微信虽然已经打开，但 Portal 还是拿不到当前页面的可访问树。

最后一段 `recommendations` 是下一步建议，优先按那里给的修复动作做，不用自己猜。

`fix_commands` 会直接列出建议执行的命令；`--fix` 只会做安全自动修复，比如 Portal 恢复，不会擅自改你的 API key 或配置。

- `小红书` 是可读的
- `微信` 先按“受限”处理，某些页面前台可见，但 Portal 仍然读不到

如果你看到的是：

```text
Portal state check failed: Accessibility service not available
```

那就不是微信命令有问题，而是手机里的 `Droidrun Portal` 无障碍服务当前没有真正启用。

先执行：

```bash

```

如果还是不行，就保持手机亮屏并解锁，进入系统“无障碍功能”页面，确认 `Droidrun Portal` 显示为“已开启”。

### 2. 为什么会出现 `FP16 is not supported on CPU; using FP32 instead`

这是本地 Whisper 在 CPU 上转写时的提示，不是报错。

含义是：

- 你的电脑现在用 CPU 跑语音识别
- CPU 不支持 FP16
- Whisper 自动改用 FP32

一般不影响使用，只是会慢一点。

## 重要教训

- 如果 DroidRun 失败，先不要急着改命令解析逻辑。
- 先确认是不是 Portal 读不到当前 UI。
- 这个项目里的本地 wrapper 只应该做“薄兜底”，不应该替代 DroidRun 自己的任务规划逻辑。
- 最优先的排查顺序是：
  - 先看有没有 `No active window or root filtered out`
  - 有的话先跑 `./fix_portal.sh`
  - Portal 恢复后，再重新执行原始命令
